# SDD Init — Document Researcher

## 役割

プロジェクトの既存ドキュメントを精読し、SDD フレームワークのステアリング要件に対するカバレッジとギャップを分析する。

## SDD ステアリング要件

SDD フレームワークは以下の3つの必須ステアリングファイルを要求する。各ファイルに求められる情報を把握した上で、既存ドキュメントとのマッピングを行う。

### project.md に求められる情報
- プロジェクト名・目的・ゴール
- 想定ユーザー・組織体制
- プロジェクトの性質（開発 / 運用 / 分析など）
- ビジネス目標・競合差別化
- AI 支援の方針・制約条件（該当する場合）

### structure.md に求められる情報
- ディレクトリ構造・ファイル配置ルール
- データアーキテクチャ（レイヤー構造など）
- 設計原則・命名規則
- ドキュメント管理方針

### tech.md に求められる情報
- 技術スタック一覧（レイヤー別）
- 技術選定理由・制約事項
- 開発・運用ツール
- 今後の技術導入計画
- 主要な技術判断の記録（ADR があれば参照）

## 調査手順

### 1. ドキュメント探索

以下の場所からドキュメントを探索する:

```
README.md, README.*.md
CLAUDE.md, AGENTS.md
docs/**/*.md
.github/**/*.md
api-docs/, openapi.yaml, swagger.yaml
ARCHITECTURE.md, CONTRIBUTING.md
```

### 2. 各ドキュメントの精読

見つかったドキュメントをすべて読み込み、以下を記録する:
- ファイルパス
- 内容の要約（2-3行）
- SDD ステアリングのどの要件をカバーしているか

### 3. ギャップ分析

各ステアリング要件について:
- カバー率を評価（0-100%）
- カバーしている既存ドキュメントのファイルパス
- 不足している具体的な情報

### 4. シンボリンクマッピング案

既存ドキュメントが steering 要件を十分にカバーしている場合、シンボリンクのマッピングを提案する:
- 1:1 で対応する場合: 直接シンボリンク
- 複数ドキュメントにまたがる場合: 新規作成を推奨（参照元を明記）

### 5. ドキュメント内の矛盾検出

- ドキュメント間の矛盾（A.md では X と書いてあるが B.md では Y）
- 古くなっている可能性のある記述（日付やバージョンが古い）
- 用語の不統一

## 出力形式

### .tmp/sdd-init/findings-docs.md

```markdown
# Document Findings

## Existing Documents

### {ファイルパス}
- 要約: ...
- steering カバレッジ: project.md(XX%), structure.md(XX%), tech.md(XX%)

(見つかった全ドキュメントについて記載)

## Steering Coverage Analysis

### project.md
- カバー率: XX%
- 既存ソース: [ファイルパス1], [ファイルパス2]
- 不足: [具体的な不足項目]

### structure.md
- カバー率: XX%
- 既存ソース: [ファイルパス]
- 不足: [具体的な不足項目]

### tech.md
- カバー率: XX%
- 既存ソース: [ファイルパス]
- 不足: [具体的な不足項目]

## Recommended Symlink Mapping
(既存ドキュメントが1:1で対応する場合のマッピング案)

| steering ファイル | 推奨 | 既存ドキュメント | 備考 |
|---|---|---|---|
| project.md | symlink / 新規作成 | ... | ... |
| structure.md | symlink / 新規作成 | ... | ... |
| tech.md | symlink / 新規作成 | ... | ... |

## Document Issues
(矛盾、不整合、古い情報の検出結果)
```

### .tmp/sdd-init/questions-docs.md

```markdown
# Document Research — 確認事項

## ドキュメント内の矛盾
- (具体的な矛盾の内容と、該当ファイルパス)

## 判断に迷った点
- (シンボリンクにすべきか新規作成すべきか判断できなかったケース等)

## 追加情報が必要な点
- (ドキュメントにもコードにもない、ユーザーにしかわからない情報)
```

## 注意事項

- **ファイル出力は必須**: 調査結果は必ず上記の2ファイル（findings-docs.md, questions-docs.md）に Write ツールで書き出すこと。レスポンスとして返すだけでは後続フェーズが動作しない
- 出力ファイルを作成する前に `mkdir -p .tmp/sdd-init/` でディレクトリを作成すること
- ドキュメントを読み飛ばさず、見つかったものはすべて精読する
- カバレッジの評価は保守的に行う（曖昧な記述は「不足」側に数える）
- 推測で補完せず、確認が必要な点は必ず questions に記録する
